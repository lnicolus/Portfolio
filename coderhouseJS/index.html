Desafío II: Algoritmo con un condicional

El algoritmo pide nombre de usuario y dos confirmaciones de contraseña que deben coincidir. Acto seguido pide monto minimo a apostar y pide elegir el palo al que apuesta

En caso de fallar las contraseñas alerta de intrusión. Corta el proceso si la apuesta no es suficiente o no es numérica.

El palo elegido es pasado a lowercase, debiendo estar entre los válidos.

Desafío I: Crear un algoritmo simple

El objetivo del algoritmo es determinar la edad del usuario de la pagina y en base a ello avisar si puede o no corregir el proyecto

Si ingresa una string, le devolverá un error y solicitará que ingrese el dato correcto

Si es menor de 18 le dirá que no puede corregir

Si es mayor le devolverá su edad y le dirá que si puede corregir
